# Reflect Context Playbook

Use this at the end of a longer session or after a frustrating task to improve
the repository's agent guidance.

## Goal

Review the current session and workspace context to assess how well the rules,
docs, playbooks, and local setup supported the work.

## Questions To Answer

1. Were the current rules and docs up to date?
2. Which rules were useful, and why?
3. What guidance was missing and would have saved time?
4. What guidance was incorrect, ambiguous, duplicated, or conflicting?
5. Which file paths should be updated?
6. What should be added, removed, or rewritten?
7. What context would have helped the agent do better?

## Output

```markdown
## State Of Rules/Docs

## What Was Useful

## What Was Missing

## What Was Incorrect Or Outdated

## Decision On Rule Changes

## Suggested Updates

## Context Improvements

## Questions
```

If the user asked for analysis only, do not edit files. If the user asked you to
improve the rules, make focused updates after the reflection.
